# 日翊盤點中心組織結構

排序即「顯示／報表／篩選」的固定順序，請勿用字母或筆畫排序，一律照下列順序。

| 部別 | 課別（依序） |
|------|--------------|
| 一部 | 北一課、北二課、北三課、北四課、桃竹課 |
| 二部 | 台中課、嘉南課、高屏課 |
| 業務部 | 業務課、訓練課 |

## 套用規則

1. **課別顯示／篩選一律照上表順序**（部別內的課別順序也固定），不可字母／筆畫排序。
2. **課別 → 部別**可由上表反查（例：北一課→一部、台中課→二部、業務課→業務部）。資料若缺「部別」，用課別推得。
3. **報表部別小計**：同部別的課別加總為該部小計，並依「一部 → 二部 → 業務部」排列；總計＝各部小計相加。
4. 新增課別／部別時，先向使用者確認要插入的**位置**（順序有意義），再更新此表。

## 可重用程式片段（JS）

本平台 `script.js` 已內建對應的 `ORG` 常數，維持與本表一致；如需在別處重用：

```js
const ORG = [
  { div: "一部", depts: ["北一課", "北二課", "北三課", "北四課", "桃竹課"] },
  { div: "二部", depts: ["台中課", "嘉南課", "高屏課"] },
  { div: "業務部", depts: ["業務課", "訓練課"] },
];
const DEPT_TO_DIV = {};
ORG.forEach((o) => o.depts.forEach((d) => { DEPT_TO_DIV[d] = o.div; }));
const DEPT_ORDER = ORG.reduce((a, o) => a.concat(o.depts), []);
const deptRank = (d) => { const i = DEPT_ORDER.indexOf(d); return i < 0 ? 999 : i; };
// 課別清單依組織順序排序：arr.sort((a,b)=>deptRank(a)-deptRank(b))
```

## 備註
- 此為日翊內部組織，僅作資料分類／排序用途，勿填入真實個資。
- 僅限本盤點平台專案使用（專案層級 skill）。
